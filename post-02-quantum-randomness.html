What Is Pseudorandomness?

Every time you call Math.random() in JavaScript, or random.random() in Python, you're running a deterministic algorithm. The most common implementation is the Mersenne Twister (MT19937), which generates 219937-1 values before repeating.

That sounds huge. And it is. For games, simulations, and most applications, it's perfectly fine. But it has a fundamental property that makes it different from true randomness:

Given the same internal state, a PRNG will always produce the same sequence. The output is predetermined the moment you set the seed. There is no uncertainty — only the illusion of it.

An attacker who observes enough output can reconstruct the internal state and predict every future value. With Mersenne Twister, this requires only 624 consecutive 32-bit outputs.

What Is Quantum Randomness?

Quantum randomness comes from the measurement of quantum states. Here's the simplest example:

  1. Take a qubit initialized to |0⟩.
  2. Apply a Hadamard gate. The qubit is now in superposition: (|0⟩ + |1⟩) / √2.
  3. Measure it.

The outcome — 0 or 1 — is governed by the Born rule. There is a 50% probability of each outcome, and no hidden variable, no internal state, no algorithm determines which one you get. This is not a gap in our knowledge. According to quantum mechanics, the outcome is fundamentally undetermined until measurement.

This is what physicists mean by "true randomness." The outcome doesn't exist before measurement. It's not hidden. It's not computed. It emerges from the measurement interaction itself.

The Key Difference: Same Seed, Same Output

Here's the clearest way to understand the distinction:

PRNG (Pseudorandom)

  • Seed: 42 → Output: 0.6394...
  • Seed: 42 → Output: 0.6394... (always)
  • Deterministic algorithm
  • Reproducible by design
  • State can be reconstructed from output

Quantum RNG (True Random)

  • Circuit run #1 → Output: 0.8271...
  • Circuit run #2 → Output: 0.1548... (different)
  • Physical measurement process
  • Non-reproducible by the laws of physics
  • No internal state to reconstruct

When we run the same quantum circuit twice on the same hardware, we get different measurement distributions. This isn't noise or error — it's the fundamental nature of quantum mechanics.

Our Hybrid Approach: Quantum Seed + Deterministic Stream

For the Quantum Genesis collection, we used a hybrid approach that combines the best of both worlds:

  1. Quantum source: A 12-qubit H+CNOT maximum-entropy circuit runs on real hardware (Origin Quantum WK_C180 or IBM Quantum ibm_fez/ibm_torino).
  2. Measurement: 8,000 shots (Origin) or 4,096 shots (IBM) produce a probability distribution.
  3. Hashing: The distribution is hashed via SHA-256 to produce a 256-bit seed.
  4. Deterministic expansion: The seed initializes a QuantumRNG class that produces a reproducible stream of values.

This is the paradox at the heart of the project:

The art is fully deterministic from its seed. But the seed itself is quantum random. You can reproduce any piece if you have its seed. But no one — not even the quantum computer — could have predicted what that seed would be before the circuit ran.

Why do it this way? Because we want verifiability. Anyone can take a quantum seed from our metadata and regenerate the exact same art. The quantum randomness provides the unpredictable origin; the deterministic RNG provides the reproducible pipeline.

The QuantumRNG Class

Here's the core of our deterministic RNG, seeded from quantum measurements:

import hashlib
import struct

class QuantumRNG:
    """Deterministic RNG seeded from quantum measurement data.
    Uses SHA-512 expansion + xorshift128+ for fast generation."""

    def __init__(self, hex_seed: str):
        # Expand the 256-bit quantum seed to 512 bits via SHA-512
        expanded = hashlib.sha512(bytes.fromhex(hex_seed)).digest()
        # Initialize two 64-bit state variables
        self.s0 = struct.unpack('<Q', expanded[0:8])[0]
        self.s1 = struct.unpack('<Q', expanded[8:16])[0]
        if self.s0 == 0:
            self.s0 = 0xDEADBEEFCAFEBABE
        if self.s1 == 0:
            self.s1 = 0x0123456789ABCDEF

    def _next(self) -> int:
        """xorshift128+ algorithm — one step."""
        s1 = self.s0
        s0 = self.s1
        self.s0 = s0
        s1 ^= (s1 << 23) & 0xFFFFFFFFFFFFFFFF
        s1 ^= (s1 >> 17)
        s1 ^= s0
        s1 ^= (s0 >> 26)
        self.s1 = s1
        return (self.s0 + self.s1) & 0xFFFFFFFFFFFFFFFF

    def random(self) -> float:
        """Return a float in [0, 1)."""
        return (self._next() >> 11) / (1 << 53)

    def randint(self, a: int, b: int) -> int:
        """Return a random integer in [a, b]."""
        return a + int(self.random() * (b - a + 1))

    def choice(self, seq):
        """Pick a random element from a sequence."""
        return seq[self.randint(0, len(seq) - 1)]

The key design decisions:

  • SHA-512 expansion ensures the full entropy of the quantum seed is distributed across the initial state.
  • xorshift128+ is fast, has a period of 2128-1, and passes standard statistical tests.
  • The class provides the same API as Python's random module, making it a drop-in replacement.

The Quantum Circuit

The seed-generation circuit is designed for maximum entropy:

# Simplified version of the quantum circuit
# 12 qubits: Hadamard on all + CNOT entanglement chain

from qiskit import QuantumCircuit

qc = QuantumCircuit(12, 12)

# Put all qubits in superposition
for i in range(12):
    qc.h(i)

# Create entanglement chain (CNOT cascade)
for i in range(11):
    qc.cx(i, i + 1)

# Measure all qubits
qc.measure(range(12), range(12))

# Run on real hardware with 4096 shots
# Each shot produces a 12-bit string
# The distribution of 4096 shots becomes the seed source

The Hadamard gates create superposition (each qubit is equally likely to be 0 or 1). The CNOT gates create entanglement (qubits become correlated in ways that have no classical analog). Together, they produce a measurement distribution with maximum Shannon entropy.

Statistical Analysis: Is It Actually Random?

Claiming "quantum randomness" means nothing without evidence. Here's how we validated the quality of our seeds:

Shannon Entropy

For a 12-qubit circuit with 4,096 possible outcomes (212), the maximum Shannon entropy is 12 bits. Our measurements consistently achieved entropy scores above 11.8 bits — within 98% of the theoretical maximum.

import math

def shannon_entropy(distribution: dict) -> float:
    """Calculate Shannon entropy of measurement distribution."""
    total = sum(distribution.values())
    entropy = 0.0
    for count in distribution.values():
        if count > 0:
            p = count / total
            entropy -= p * math.log2(p)
    return entropy

Chi-Squared Uniformity Test

A chi-squared test checks whether the measurement outcomes are uniformly distributed. Under the null hypothesis (uniform distribution), each of the 4,096 outcomes should appear once in 4,096 shots.

Across our 100 quantum seeds, the chi-squared p-values were consistently above 0.05, confirming that the distributions are statistically indistinguishable from uniform random. This is exactly what quantum mechanics predicts for a max-entropy circuit.

Bit-Level Analysis

We also verified that individual bits showed no bias:

  • Each bit position: approximately 50/50 split between 0 and 1
  • No detectable correlation between bit positions after the CNOT chain
  • No detectable patterns across consecutive seeds

Why This Matters for NFTs

Most generative NFT projects use block.timestamp, block.difficulty, or off-chain PRNGs to generate traits. These are all deterministic. A miner can manipulate block variables. A developer can choose seeds that produce desirable outcomes.

With quantum-sourced seeds, the provenance is different:

  • Provable uniqueness: Each seed comes from a physical event that cannot be replicated or predicted.
  • No developer manipulation: The developer cannot preview and cherry-pick favorable seeds. The quantum measurement produces what it produces.
  • Verifiable pipeline: Given the quantum seed (stored in IPFS metadata), anyone can regenerate the art and verify it matches.
  • Two-processor provenance: NFTs #1-18 carry Origin Quantum heritage; #19-100 carry IBM Quantum heritage. Each piece records which processor and backend generated its seed.

The randomness isn't just "better" — it's a fundamentally different category. PRNG outputs are computed. Quantum outputs are measured. That distinction matters when you're creating provably unique digital art.
