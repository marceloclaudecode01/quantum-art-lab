1. What Is Pinata and Why IPFS?

IPFS (InterPlanetary File System) is a decentralized protocol for storing and sharing files. Unlike HTTP, where content lives at a specific server address, IPFS uses content addressing — each file gets a unique hash (CID) based on its contents. If the file changes, the CID changes. This makes IPFS perfect for NFTs because it guarantees the artwork linked to your token can never be silently swapped.

The catch: IPFS nodes only keep files they've explicitly "pinned." Without pinning, your files get garbage-collected and disappear. Pinata is a pinning service that ensures your files stay available 24/7. They run IPFS nodes that permanently store your content and serve it through fast gateways.

Alternatives exist — nft.storage, Infura IPFS, web3.storage — but Pinata has the best developer experience and the most straightforward API. For Quantum Genesis, we uploaded all 100 PNG artworks and 100 metadata JSON files through Pinata, and they've been reliably accessible ever since.

3. Account Setup and API Keys

Head to app.pinata.cloud and create a free account. Once logged in:

  1. Go to API Keys in the sidebar
  2. Click New Key
  3. Enable pinFileToIPFS and pinJSONToIPFS permissions (or just grant Admin access for simplicity)
  4. Name the key something descriptive like "quantum-genesis-upload"
  5. Click Create Key

You'll get three values: API Key, API Secret, and JWT. Save all three immediately — the secret is only shown once. For our Python script, we'll use the JWT (it's the simplest auth method).

# Store as environment variables (never hardcode!)
# Windows:
setx PINATA_JWT "your_jwt_token_here"
setx PINATA_API_KEY "your_api_key"
setx PINATA_SECRET_KEY "your_api_secret"

# Linux/Mac:
export PINATA_JWT="your_jwt_token_here"

You'll also get a dedicated gateway — a URL like https://your-name.mypinata.cloud. This is your personal IPFS gateway for fast, reliable access to your pinned content. Ours is https://maroon-bright-perch-405.mypinata.cloud.

4. Uploading a Single File

Let's start simple. To upload one file via the Pinata API:

import requests
import os

PINATA_JWT = os.environ["PINATA_JWT"]

def upload_file(filepath):
    url = "https://api.pinata.cloud/pinning/pinFileToIPFS"
    headers = {"Authorization": f"Bearer {PINATA_JWT}"}

    filename = os.path.basename(filepath)
    with open(filepath, "rb") as f:
        files = {"file": (filename, f)}
        response = requests.post(url, headers=headers, files=files)

    data = response.json()
    print(f"Uploaded {filename}")
    print(f"CID: {data['IpfsHash']}")
    print(f"Size: {data['PinSize']} bytes")
    return data["IpfsHash"]

# Usage
cid = upload_file("quantum_genesis_001.png")
# Access at: https://your-gateway.mypinata.cloud/ipfs/{cid}

The response includes the IpfsHash (the CID), the PinSize, and a Timestamp. The CID is what you'll reference in your NFT metadata.

5. Uploading a Folder (Our 100 SVGs → PNGs)

For NFT collections, you want all images in a single IPFS directory. This way, individual images are accessible at ipfs://CID/filename.png, which is the standard pattern for NFT metadata.

import requests
import os

PINATA_JWT = os.environ["PINATA_JWT"]

def upload_folder(folder_path, pin_name="nft-art"):
    url = "https://api.pinata.cloud/pinning/pinFileToIPFS"
    headers = {"Authorization": f"Bearer {PINATA_JWT}"}

    files = []
    for filename in sorted(os.listdir(folder_path)):
        filepath = os.path.join(folder_path, filename)
        if os.path.isfile(filepath):
            # The tuple format: (form field, (path_in_ipfs, file_obj, mime))
            files.append((
                "file",
                (f"{pin_name}/{filename}", open(filepath, "rb"), "image/png")
            ))

    # Metadata to name the pin
    import json
    metadata = json.dumps({"name": pin_name})
    options = json.dumps({"wrapWithDirectory": False})

    response = requests.post(
        url,
        headers=headers,
        files=files,
        data={"pinataMetadata": metadata, "pinataOptions": options}
    )

    data = response.json()
    print(f"Folder uploaded! CID: {data['IpfsHash']}")

    # Close all file handles
    for _, (_, fobj, _) in files:
        fobj.close()

    return data["IpfsHash"]

# Upload all 100 PNGs
art_cid = upload_folder("./nft-output/png/", pin_name="quantum-genesis-art")
# Each image: ipfs://{art_cid}/quantum_genesis_001.png

6. Uploading Metadata JSONs

NFT metadata follows the ERC-721 metadata standard. Each token needs a JSON file with at minimum a name, description, and image field. Here's what one of ours looks like:

{
  "name": "Quantum Genesis #42",
  "description": "Generated from quantum measurements on IBM Quantum ibm_fez (156 qubits).",
  "image": "ipfs://bafybeifges7tei5x7drj37f34yhzqofwlz2icbo7z67isg6g446k65yw3a/quantum_genesis_042.png",
  "attributes": [
    {"trait_type": "Processor", "value": "IBM Quantum ibm_fez"},
    {"trait_type": "Qubits Used", "value": 8},
    {"trait_type": "Entropy Tier", "value": "High"},
    {"trait_type": "Entanglement", "value": "GHZ-3"},
    {"trait_type": "Color Palette", "value": "Nebula"},
    {"trait_type": "Complexity Score", "value": 87}
  ]
}

Notice the image field uses ipfs:// protocol, not an HTTP gateway URL. This is important — marketplaces like OpenSea resolve ipfs:// links through their own gateways, ensuring long-term availability.

Upload the metadata folder the same way as the art folder:

metadata_cid = upload_folder("./nft-output/metadata/", pin_name="quantum-genesis-metadata")
# Each metadata: ipfs://{metadata_cid}/42

The metadata CID becomes your smart contract's baseTokenURI. When OpenSea calls tokenURI(42), your contract returns ipfs://{metadata_cid}/42, OpenSea fetches the JSON, reads the image field, and displays your art.

7. Understanding CIDs

A CID (Content Identifier) is a cryptographic hash of your file's contents. Two important properties:

  • Deterministic: The same file always produces the same CID. Upload the same PNG twice and you get the same hash.
  • Immutable: If you change even one byte, the CID changes completely. This is why CIDs are perfect for NFT provenance — you can mathematically prove the artwork hasn't been tampered with.

CIDs come in two versions:

VersionPrefixExample
CIDv0Qm...QmT5NvUtoM5nWFfrQdVrFtvGfKFmG7AHE8P34isapyhCxX
CIDv1bafy...bafybeifges7tei5x7drj37f34yhzqofwlz2icbo7z67isg6g446k65yw3a

Pinata returns CIDv0 by default for single files and CIDv1 for directories. Both work fine. CIDv1 is the modern standard and is compatible with subdomains, which matters for gateway URLs.

8. Verifying on Gateway

After uploading, always verify your files are accessible. You have several gateway options:

# Your dedicated Pinata gateway (fastest)
https://your-name.mypinata.cloud/ipfs/{CID}/filename.png

# Public IPFS gateway (slower, sometimes down)
https://ipfs.io/ipfs/{CID}/filename.png

# Cloudflare IPFS gateway
https://cloudflare-ipfs.com/ipfs/{CID}/filename.png

A quick verification script:

import requests

def verify_uploads(cid, filenames, gateway):
    for fname in filenames:
        url = f"{gateway}/ipfs/{cid}/{fname}"
        r = requests.head(url, timeout=10)
        status = "OK" if r.status_code == 200 else f"FAIL ({r.status_code})"
        print(f"{fname}: {status}")

verify_uploads(
    "bafybeifges7tei5x7drj37f34yhzqofwlz2icbo7z67isg6g446k65yw3a",
    [f"quantum_genesis_{i:03d}.png" for i in range(1, 101)],
    "https://maroon-bright-perch-405.mypinata.cloud"
)

9. Complete Python Batch Upload Script

Here's the production script we used for Quantum Genesis. It handles retries, progress tracking, and outputs the CID mapping you'll need for metadata generation:

#!/usr/bin/env python3
"""
upload_to_ipfs.py — Batch upload NFT art and metadata to Pinata IPFS
Used for the Quantum Genesis collection (100 NFTs)
"""

import os
import json
import time
import requests
from pathlib import Path

PINATA_JWT = os.environ["PINATA_JWT"]
PINATA_API = "https://api.pinata.cloud/pinning/pinFileToIPFS"
HEADERS = {"Authorization": f"Bearer {PINATA_JWT}"}
MAX_RETRIES = 3

def upload_directory(folder_path, pin_name, file_ext=None):
    """Upload a directory of files as a single IPFS directory pin."""
    folder = Path(folder_path)
    files_to_upload = sorted(folder.iterdir())

    if file_ext:
        files_to_upload = [f for f in files_to_upload if f.suffix == file_ext]

    print(f"Uploading {len(files_to_upload)} files from {folder_path}...")

    file_handles = []
    files = []
    for filepath in files_to_upload:
        fh = open(filepath, "rb")
        file_handles.append(fh)
        mime = "image/png" if filepath.suffix == ".png" else "application/json"
        files.append(("file", (f"{pin_name}/{filepath.name}", fh, mime)))

    metadata = json.dumps({"name": pin_name})
    options = json.dumps({"wrapWithDirectory": False})

    for attempt in range(MAX_RETRIES):
        try:
            # Reset file positions on retry
            for fh in file_handles:
                fh.seek(0)

            response = requests.post(
                PINATA_API,
                headers=HEADERS,
                files=files,
                data={"pinataMetadata": metadata, "pinataOptions": options},
                timeout=300  # 5 min timeout for large uploads
            )
            response.raise_for_status()
            data = response.json()

            print(f"SUCCESS! CID: {data['IpfsHash']}")
            print(f"Pin size: {data['PinSize']} bytes")

            for fh in file_handles:
                fh.close()
            return data["IpfsHash"]

        except Exception as e:
            print(f"Attempt {attempt+1} failed: {e}")
            if attempt < MAX_RETRIES - 1:
                time.sleep(5 * (attempt + 1))

    for fh in file_handles:
        fh.close()
    raise Exception(f"Failed after {MAX_RETRIES} retries")


def generate_metadata(art_cid, nft_data_path, output_dir):
    """Generate ERC-721 metadata JSONs pointing to the art CID."""
    os.makedirs(output_dir, exist_ok=True)

    with open(nft_data_path) as f:
        nfts = json.load(f)

    for nft in nfts:
        token_id = nft["token_id"]
        metadata = {
            "name": f"Quantum Genesis #{token_id}",
            "description": nft.get("description", ""),
            "image": f"ipfs://{art_cid}/{nft['filename']}",
            "attributes": nft.get("attributes", [])
        }

        outpath = os.path.join(output_dir, str(token_id))
        with open(outpath, "w") as f:
            json.dump(metadata, f, indent=2)

    print(f"Generated {len(nfts)} metadata files in {output_dir}")


if __name__ == "__main__":
    # Step 1: Upload art
    art_cid = upload_directory("./nft-output/png/", "quantum-genesis-art", ".png")

    # Step 2: Generate metadata pointing to art CID
    generate_metadata(art_cid, "./_nft_data.json", "./nft-output/metadata/")

    # Step 3: Upload metadata
    meta_cid = upload_directory("./nft-output/metadata/", "quantum-genesis-metadata")

    print(f"\n{'='*60}")
    print(f"Art CID:      {art_cid}")
    print(f"Metadata CID: {meta_cid}")
    print(f"Base URI:     ipfs://{meta_cid}/")
    print(f"{'='*60}")
    print(f"\nSet this as your contract's baseTokenURI:")
    print(f"  contract.setBaseTokenURI('ipfs://{meta_cid}/')")

10. Tips for Large Collections

After uploading 100 NFTs, here's what we learned:

  • Always upload as directories. Individual file pins eat your 500-file quota fast. A directory of 100 files counts as 1 pin.
  • Use PNG, not SVG. OpenSea and most marketplaces display PNGs reliably. SVGs sometimes render incorrectly or don't display at all. We generated our art as SVG, then converted to PNG with Selenium (because cairosvg doesn't handle our complex SVGs well on Windows).
  • Upload art first, then metadata. Metadata references the art CID, so you need the art CID before generating metadata JSONs.
  • Set a long timeout. Uploading 100 PNGs (15MB+) can take a few minutes. Our script uses a 5-minute timeout.
  • Verify every file after upload. Run HEAD requests against your gateway for each file. We found 0 failures, but it's worth checking.
  • Use ipfs:// in metadata, not HTTP gateway URLs. The ipfs:// protocol is the standard. Marketplaces resolve it through their own infrastructure.
  • Keep your CIDs safe. Write them down, save them in env vars, commit them to your repo. Losing a CID means losing the link between your token and its content.
  • Consider a paid plan for 1000+ collections. The free tier works great for small-to-medium collections, but if you're doing a 10K PFP project, you'll want the Picnic or Submarine plan.

Pinata makes IPFS painless. For Quantum Genesis, the entire upload process — 100 PNGs and 100 metadata files — took about 10 minutes and cost exactly $0. The content has been reliably pinned and accessible through our gateway ever since.
