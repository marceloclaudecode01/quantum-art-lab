Prerequisites

  • Python 3.9+ installed on your system
  • A free IBM Quantum account at quantum.ibm.com
  • Basic Python knowledge (functions, imports, dictionaries)
  • About 15 minutes of setup time and 1-5 minutes of queue wait time per job
What you will build: A Python script that connects to a real IBM quantum processor, runs a 12-qubit circuit, collects measurement data, and converts it into a deterministic hex seed you can use for any generative art pipeline.

Step-by-Step Tutorial

  1. Install Qiskit

    You need two packages: qiskit (the core SDK) and qiskit-ibm-runtime (the interface to IBM's cloud quantum processors).

    pip install qiskit qiskit-ibm-runtime

    This installs Qiskit 1.x with the Runtime primitives API. If you have an older version of Qiskit installed, upgrade with pip install --upgrade qiskit qiskit-ibm-runtime.

  2. Get Your IBM Quantum Token

    Go to quantum.ibm.com and sign up for a free account. Once logged in, navigate to your account settings and copy your API token.

    The free tier gives you access to real quantum processors with fair-share queuing. No credit card required. Processors available include ibm_fez (156 qubits), ibm_torino (133 qubits), ibm_marrakesh (156 qubits), and ibm_kingston (156 qubits).

    Store the token as an environment variable rather than hardcoding it:

    # Linux/Mac
export IBM_QUANTUM_TOKEN="your_token_here"

# Windows PowerShell
$env:IBM_QUANTUM_TOKEN = "your_token_here"

  3. Initialize the Service

    import os
from qiskit_ibm_runtime import QiskitRuntimeService, SamplerV2 as Sampler

token = os.environ["IBM_QUANTUM_TOKEN"]
service = QiskitRuntimeService(
    channel="ibm_quantum_platform",
    token=token
)

    The channel parameter must be "ibm_quantum_platform". This is the current channel name — older tutorials may show "ibm_quantum" which no longer works.

  4. Select a Backend

    Instead of hardcoding a specific processor, we query for all available real backends and pick the one with the shortest queue:

    backends = service.backends(
    simulator=False,
    operational=True,
    min_num_qubits=8
)
backends.sort(key=lambda b: b.status().pending_jobs)
backend = backends[0]
print(f"Selected backend: {backend.name} ({backend.num_qubits} qubits, "
      f"{backend.status().pending_jobs} pending jobs)")

    This approach is resilient. If ibm_fez is under maintenance, the script automatically picks the next best option. For Quantum Genesis, most jobs landed on ibm_fez (156 qubits) and ibm_torino (133 qubits).

  5. Build the Quantum Circuit

    This is the core of the seed generation — a 12-qubit circuit designed for maximum entropy:

    from qiskit import QuantumCircuit

def build_nft_seed_circuit(n_qubits=12):
    """H + alternating CNOT max-entropy seed-generation circuit."""
    qc = QuantumCircuit(n_qubits, n_qubits)

    # Hadamard layer: put every qubit into superposition
    for i in range(n_qubits):
        qc.h(i)

    # Even CNOT layer: entangle pairs (0,1), (2,3), (4,5)...
    for i in range(0, n_qubits - 1, 2):
        qc.cx(i, i + 1)

    # Odd CNOT layer: entangle pairs (1,2), (3,4), (5,6)...
    for i in range(1, n_qubits - 1, 2):
        qc.cx(i, i + 1)

    # Measure all qubits
    qc.measure(range(n_qubits), range(n_qubits))
    return qc

circuit = build_nft_seed_circuit()

    The Hadamard gates create superposition. The alternating CNOT layers create entanglement — correlations between qubits that cannot exist classically. When measured, these correlations produce probability distributions that no pseudorandom generator can replicate.

  6. Transpile and Run

    Real quantum processors have specific hardware constraints — not every qubit can connect to every other qubit. Transpilation adapts your circuit to the hardware's topology:

    from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

# Transpile for the selected backend
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_circuit = pm.run(circuit)

# Run on real hardware using SamplerV2
sampler = Sampler(mode=backend)
job = sampler.run([isa_circuit], shots=4096)
print(f"Job ID: {job.job_id()} — waiting for results...")
result = job.result()

    The optimization_level=1 balances compilation speed with circuit optimization. Higher levels (2 or 3) reduce gate count but take longer to compile. For seed generation, level 1 is sufficient.

    Wait time depends on queue depth — typically 30 seconds to 5 minutes on the free tier. The job.result() call blocks until the job completes.

  7. Extract Counts and Generate Seed

    SamplerV2 returns results in a specific format. The classical register is accessed via the c attribute (not meas — this is a common mistake):

    import hashlib

# Extract measurement counts from SamplerV2 result
pub_result = result[0]
counts = pub_result.data.c.get_counts()

# Convert counts to deterministic hex seed
sorted_counts = sorted(counts.items())
raw = "|".join(f"{k}:{v}" for k, v in sorted_counts)
seed = hashlib.sha256(raw.encode()).hexdigest()
print(f"Quantum seed: {seed}")
# Example: "a3f7c1d8e5b2094f6..."  (64-char hex string)

    The SHA-256 hash ensures a deterministic mapping from measurement data to seed. Same counts always produce the same seed, but even a single bit flip in the quantum measurement produces a completely different hash.

    Why SHA-256? Raw measurement counts are noisy and variable-length. Hashing normalizes them into a fixed 256-bit value with uniform distribution — perfect for seeding any PRNG or art generator.

  8. Feed the Seed Into Your Art Generator

    The 64-character hex seed can drive any generative art pipeline. Here is a minimal example using Python's random module:

    import random

# Use the quantum seed for deterministic art generation
rng = random.Random(seed)

# Generate art parameters from quantum randomness
palette_index = rng.randint(0, 15)
symmetry = rng.choice(["radial", "bilateral", "asymmetric"])
complexity = rng.uniform(0.3, 1.0)
hue_base = rng.randint(0, 360)

print(f"Palette: {palette_index}, Symmetry: {symmetry}")
print(f"Complexity: {complexity:.2f}, Hue: {hue_base}")

# Pass these parameters to your SVG/Canvas/PIL renderer...

    For Quantum Genesis, the seed drove every visual attribute: qubit configuration (GHZ-3, Steane-7, etc.), color palette, geometric structure, and fine detail. Each NFT's metadata records the exact quantum backend and shot count used.

Complete Working Code

Here is everything combined into a single, runnable script under 50 lines:

"""Quantum random seed generator for generative art.
Connects to IBM Quantum, runs a 12-qubit circuit, returns a hex seed."""

import os
import hashlib
from qiskit import QuantumCircuit
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService, SamplerV2 as Sampler

def build_nft_seed_circuit(n_qubits=12):
    qc = QuantumCircuit(n_qubits, n_qubits)
    for i in range(n_qubits):
        qc.h(i)
    for i in range(0, n_qubits - 1, 2):
        qc.cx(i, i + 1)
    for i in range(1, n_qubits - 1, 2):
        qc.cx(i, i + 1)
    qc.measure(range(n_qubits), range(n_qubits))
    return qc

def generate_quantum_seed():
    # Connect to IBM Quantum
    service = QiskitRuntimeService(
        channel="ibm_quantum_platform",
        token=os.environ["IBM_QUANTUM_TOKEN"]
    )

    # Pick the least-busy real backend
    backends = service.backends(simulator=False, operational=True, min_num_qubits=8)
    backends.sort(key=lambda b: b.status().pending_jobs)
    backend = backends[0]
    print(f"Backend: {backend.name} ({backend.num_qubits}q)")

    # Build, transpile, and run
    circuit = build_nft_seed_circuit()
    pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
    isa_circuit = pm.run(circuit)

    sampler = Sampler(mode=backend)
    job = sampler.run([isa_circuit], shots=4096)
    result = job.result()

    # Extract counts and hash into seed
    counts = result[0].data.c.get_counts()
    sorted_counts = sorted(counts.items())
    raw = "|".join(f"{k}:{v}" for k, v in sorted_counts)
    seed = hashlib.sha256(raw.encode()).hexdigest()
    print(f"Quantum seed: {seed}")
    return seed

if __name__ == "__main__":
    generate_quantum_seed()

Save this as quantum_seed.py, set your IBM_QUANTUM_TOKEN environment variable, and run it. You will get a 64-character hex seed derived from a real quantum measurement in about 1-5 minutes.

Common Pitfalls

Wrong channel name

Use "ibm_quantum_platform", not "ibm_quantum". The channel name changed in recent Qiskit Runtime versions. Using the old name raises an authentication error.

SamplerV2 result format

The classical register is accessed via result[0].data.c, where c is the default classical register name. If you named your classical register something else in the circuit, use that name instead. The old .data.meas accessor no longer works.

Transpilation is required

You cannot run an abstract circuit directly on hardware. The generate_preset_pass_manager handles gate decomposition, qubit routing, and optimization for the specific processor. Skipping this step raises an error.

Queue times vary

Free-tier jobs share processor time fairly. During peak hours, you may wait 5-10 minutes. The backend selection logic (sorting by pending jobs) minimizes this. For batch generation, submit multiple jobs and collect results asynchronously.

Shots affect seed quality

More shots = more measurement data = richer probability information. We used 4096 shots (IBM's default). Using fewer than 1000 shots produces less statistically significant distributions. For art, 4096 is a good balance between quality and job time.
